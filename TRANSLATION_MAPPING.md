## Translation Key Naming Conventions

### Pattern: Article X Content
- `aX_lead`: Article X lead/opening paragraph
- `aX_pN`: Paragraph N in article X (numbered sequentially through content)
- `aX_figN_caption`: Figure N caption
- `aX_kfN`: Key finding N in article X
- `aX_conclusion`: Conclusion section

### Pattern: Corporate Solutions Content
- `corp_value_pN`: Value proposition paragraph N
- `corp_framework_intro_p`: Framework introduction paragraph
- `corp_clusterN_desc`: Description for cluster N
- `corp_application_pN`: Application section paragraph N
- `corp_impactN_h4`: Impact item N heading
- `corp_impactN_p`: Impact item N description
- `corp_cta_p`: Call-to-action paragraph

### Shared Keys
- `shared_key_finding`: "Key finding" label used across all articles

## HTML Integration Notes

### For Articles (article1.html, article2.html, article3.html)
To use these translations, add `data-i18n` attributes to the relevant elements:

```html
<p data-i18n="a1_p1">Senior leaders rarely fail for lack...</p>
<p class="figure-caption"><strong>Reading the model:</strong> <span data-i18n="a1_fig1_caption">Below the competence threshold...</span></p>
```

### For Corporate Solutions (corporate-solutions.html)
Similar pattern with `data-i18n` attributes:

```html
<p data-i18n="corp_value_p1">Track career velocity...</p>

  <p class="corp-cluster-text" data-i18n="corp_cluster1_desc">How AI disruption and digital transformation...</p>
</div>
```
